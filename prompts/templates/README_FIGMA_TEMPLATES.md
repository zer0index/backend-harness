## ‚ö†Ô∏è Critical Points

### **1. Figma Make CANNOT Access Localhost**

**Problem:**
```typescript
// ‚ùå This will FAIL in Figma Make
fetch('http://localhost:8000/api/v1/users')
```

**Solution:**
```typescript
// ‚úÖ Use mock data in Figma Make
import users from './mock-data/users.json';
```

**Guideline addresses:** Detailed patterns for mock data development and API swapping

---

### **2. Brainstorming = Frontend Features, NOT Backend Endpoints**

**Wrong:**
> "We could add a new analytics endpoint to calculate user statistics..."

**Right:**
> "Using the existing user data with timestamps, we could build an activity dashboard that calculates statistics in the frontend..."

**Prompt addresses:** Clear examples of right vs. wrong brainstorming approach

---

### **3. Data Model MUST Match Backend Exactly**

**Problem:**
```typescript
// ‚ùå Backend uses snake_case
interface Task {
  userId: number;
}
```

**Solution:**
```typescript
// ‚úÖ Match backend exactly
interface Task {
  user_id: number;
}
```

**Guideline addresses:** Strict data model adherence with examples

## üéì Example Workflow

### **Scenario: Task Management API**

**Step 1: Generate Backend**
```bash
python autonomous_agent_demo.py --project-dir ./task_api --config medium
```

**Initializer agent automatically creates:**
- ‚úÖ Complete FastAPI backend
- ‚úÖ `docs/APP_OVERVIEW.md` (business context)
- ‚úÖ `docs/FRONTEND_HANDOFF.md` (technical guide)
- ‚úÖ `docs/FIGMA_MAKE_PROMPT.md` (customized kickoff - ready to use!)
- ‚úÖ `docs/mock-data/*.json` (realistic sample data)
- ‚úÖ `guidelines/guideline.md` (universal coding standards)

**Step 2: Create Figma Make Project**
- Name: "Task Management Frontend"
- Framework: React + TypeScript

**Step 3: Copy Guideline**
```bash
# Copy auto-generated guideline to Figma Make project
cp task_api/guidelines/guideline.md [figma-project]/guidelines/guideline.md
```

**Step 4: Upload Handoff Docs**
- Upload `task_api/docs/APP_OVERVIEW.md`
- Upload `task_api/docs/FRONTEND_HANDOFF.md`
- Upload `task_api/docs/mock-data/*`

**Step 5: Use Auto-Generated Prompt**

Open `task_api/docs/FIGMA_MAKE_PROMPT.md` - it's already customized with:

```markdown
# Frontend Development Kickoff - Task Management App

## Entities in This Project
- User (id, email, name, role)
- Task (id, title, description, status, priority, user_id, created_at, due_date)
- Comment (id, content, task_id, user_id, created_at)
- Tag (id, name, color)

## Key Workflows (from APP_OVERVIEW.md)
1. User creates task with title, description, status, priority
2. User assigns task to themselves or team members
3. User adds comments to tasks for collaboration
4. User filters tasks by status, priority, and tags
5. User views dashboard with task statistics

## Data Constraints (from Pydantic schemas)
- TaskStatus: "todo" | "in_progress" | "done"
- TaskPriority: "low" | "medium" | "high" | "urgent"
- User.role: "user" | "admin"
```

**Step 6: Start Conversation**
- Copy content from `task_api/docs/FIGMA_MAKE_PROMPT.md`
- Paste into Figma Make chat
- AI reviews docs
- Brainstorm features together
- Build iteratively with mock data

**Step 7: Download & Integrate**
- Download generated code from Figma Make
- Toggle `USE_MOCK_DATA = false` in config
- Connect to real backend at `http://localhost:8000`
- Test integration
